# Requirements Document: AstraMark $100M SaaS Frontend

## Introduction

This document specifies the requirements for transforming AstraMark from a basic UI into a premium, conversion-driven SaaS product designed to achieve $100M revenue. The system focuses on creating an addictive user experience that drives conversions, retention, and upgrades through instant gratification, automation, and results-oriented design.

The core principle: Users care about business growth, not features. Every interaction must answer: "Will this grow my business?"

## Glossary

- **Landing_Page**: The public-facing entry point that converts visitors into users
- **Onboarding_Flow**: The guided setup process that delivers the first "WOW moment"
- **Dashboard**: The primary user interface showing growth metrics and action panels
- **Auto_Mode**: Premium feature where AI handles content, ads, and emails automatically
- **Pricing_Page**: The subscription tier selection interface
- **Growth_Metrics**: Quantifiable business results (leads, revenue, engagement)
- **Action_Panel**: Interface section for triggering AI-powered business actions
- **Live_Feed**: Real-time display of AI actions and system activity
- **AI_Insights**: Actionable recommendations generated by the system
- **WOW_Moment**: The instant when users see tangible value (strategy, plan, or content)
- **Conversion_Machine**: Design pattern optimized for visitor-to-customer conversion
- **Design_System**: Consistent visual language (colors, typography, components, animations)
- **Business_Setup**: Initial configuration capturing business type, audience, and goals
- **AI_Analysis**: Processing phase that generates personalized strategies
- **Instant_Output**: Immediate delivery of strategy, growth plan, or content
- **Growth_Score**: Gamification metric tracking user progress and engagement
- **Micro_Interaction**: Small animated feedback for user actions
- **Glassmorphism**: Visual style using frosted glass effect with transparency
- **Hero_Section**: Top section of landing page with primary value proposition
- **Social_Proof**: Trust indicators (testimonials, metrics, logos)
- **Urgency_Trigger**: Design element creating time-sensitive motivation
- **One_Click_Action**: Interface pattern requiring single interaction to complete task

## Requirements

### Requirement 1: Landing Page Conversion System

**User Story:** As a visitor, I want to immediately understand how AstraMark will grow my business, so that I can decide to sign up within seconds.

#### Acceptance Criteria

1. THE Landing_Page SHALL display a hero section with the headline "AI That Brings You Customers Automatically"
2. THE Landing_Page SHALL present content in the sequence: Problem → Solution → Demo → Social_Proof → Pricing_Page
3. WHEN a visitor views the Landing_Page, THE Landing_Page SHALL display an actual Dashboard demo showing Growth_Metrics
4. THE Landing_Page SHALL include Social_Proof elements (testimonials, success metrics, customer logos)
5. THE Landing_Page SHALL provide a primary call-to-action button visible above the fold
6. WHEN a visitor clicks the primary call-to-action, THE Landing_Page SHALL navigate to the Onboarding_Flow
7. THE Landing_Page SHALL load and render within 2 seconds on standard broadband connections
8. THE Landing_Page SHALL use the Design_System color palette (Indigo/Purple gradient, dark background, neon accents)

### Requirement 2: Three-Step Onboarding Flow

**User Story:** As a new user, I want to complete setup quickly and see immediate results, so that I understand the value before investing time.

#### Acceptance Criteria

1. THE Onboarding_Flow SHALL consist of exactly three steps: Business_Setup, AI_Analysis, and Instant_Output
2. WHEN a user begins onboarding, THE Onboarding_Flow SHALL display step 1 (Business_Setup) requesting business type, target audience, and goals
3. WHEN a user completes Business_Setup, THE Onboarding_Flow SHALL display step 2 (AI_Analysis) with animated progress indicators
4. THE AI_Analysis step SHALL complete within 10 seconds
5. WHEN AI_Analysis completes, THE Onboarding_Flow SHALL display step 3 (Instant_Output) showing strategy, growth plan, and first content piece
6. THE Instant_Output SHALL present results in a visually impressive format using the Design_System
7. WHEN a user completes Onboarding_Flow, THE system SHALL navigate to the Dashboard
8. THE Onboarding_Flow SHALL allow users to skip optional fields and proceed with minimal input
9. THE Onboarding_Flow SHALL save progress automatically after each step

### Requirement 3: Results-Driven Dashboard

**User Story:** As a user, I want to see my business growth at a glance and take actions that drive results, so that I stay engaged and see continuous value.

#### Acceptance Criteria

1. THE Dashboard SHALL display a top section titled "Your Growth This Week" showing Growth_Metrics (leads, revenue, engagement)
2. THE Dashboard SHALL present Growth_Metrics as numerical values with percentage change indicators
3. THE Dashboard SHALL include an Action_Panel with buttons: "Generate Content", "Run Ads", "Optimize Campaign"
4. WHEN a user clicks an Action_Panel button, THE Dashboard SHALL execute the action and display results within 5 seconds
5. THE Dashboard SHALL display a Live_Feed showing real-time AI actions in chronological order
6. THE Dashboard SHALL update the Live_Feed automatically when new AI actions occur
7. THE Dashboard SHALL display an AI_Insights section with 3-5 actionable recommendations
8. WHEN a user clicks an AI_Insights recommendation, THE Dashboard SHALL execute the recommended action
9. THE Dashboard SHALL use card-based layout with rounded corners, glassmorphism effects, and soft shadows
10. THE Dashboard SHALL display a Growth_Score prominently in the header
11. THE Dashboard SHALL refresh Growth_Metrics automatically every 60 seconds

### Requirement 4: Auto Mode Premium Feature

**User Story:** As a paying user, I want AI to handle my marketing automatically, so that I can focus on my business while growth happens in the background.

#### Acceptance Criteria

1. THE Dashboard SHALL display an "Enable Auto Growth Mode" toggle in a prominent position
2. WHEN Auto_Mode is disabled, THE toggle SHALL display as off with explanatory text
3. WHEN a user clicks the Auto_Mode toggle, THE system SHALL check subscription tier
4. IF the user has a Pro or Growth subscription, THEN THE system SHALL enable Auto_Mode
5. IF the user has a Starter subscription, THEN THE system SHALL display an upgrade prompt
6. WHEN Auto_Mode is enabled, THE system SHALL automatically generate content daily
7. WHEN Auto_Mode is enabled, THE system SHALL automatically manage ad campaigns
8. WHEN Auto_Mode is enabled, THE system SHALL automatically send email sequences
9. THE Dashboard SHALL display Auto_Mode status with a visual indicator (animated when active)
10. WHEN Auto_Mode performs an action, THE Live_Feed SHALL display the action with an "Auto" badge
11. THE system SHALL allow users to disable Auto_Mode at any time

### Requirement 5: High-Conversion Pricing Page

**User Story:** As a potential customer, I want to clearly understand pricing tiers and choose the right plan, so that I can upgrade with confidence.

#### Acceptance Criteria

1. THE Pricing_Page SHALL display exactly three tiers: Starter (₹1999), Pro (₹3999), Growth (₹9999)
2. THE Pricing_Page SHALL mark the Pro tier as "Most Popular" with a visual badge
3. THE Pricing_Page SHALL list features for each tier in a comparison format
4. THE Pricing_Page SHALL highlight value propositions (results, not features) for each tier
5. THE Pricing_Page SHALL display Social_Proof elements (customer count, success rate)
6. THE Pricing_Page SHALL include Urgency_Triggers (limited-time offers, countdown timers)
7. WHEN a user selects a tier, THE Pricing_Page SHALL navigate to payment processing
8. THE Pricing_Page SHALL display annual pricing option with savings percentage
9. THE Pricing_Page SHALL use the Design_System with emphasis on the Pro tier
10. THE Pricing_Page SHALL load within 1.5 seconds

### Requirement 6: Design System Implementation

**User Story:** As a user, I want a visually stunning and consistent interface, so that the product feels premium and trustworthy.

#### Acceptance Criteria

1. THE Design_System SHALL define a color palette: Indigo (#6366F1) to Purple (#A855F7) gradient, dark background (#0F172A), neon accents (#22D3EE)
2. THE Design_System SHALL use rounded corners (border-radius: 12px) for all cards and containers
3. THE Design_System SHALL apply glassmorphism effects (backdrop-blur, transparency) to elevated components
4. THE Design_System SHALL use soft shadows (box-shadow with 20px blur, 10% opacity) for depth
5. THE Design_System SHALL define typography scale using system fonts with clear hierarchy
6. THE Design_System SHALL include Micro_Interactions for all interactive elements (buttons, toggles, inputs)
7. THE Design_System SHALL use smooth transitions (300ms ease-in-out) for state changes
8. THE Design_System SHALL implement loading skeletons for async content
9. THE Design_System SHALL ensure all components meet WCAG 2.1 AA contrast requirements
10. THE Design_System SHALL provide dark mode as the default theme

### Requirement 7: Animation and Motion System

**User Story:** As a user, I want smooth, delightful animations that make the interface feel alive, so that I enjoy using the product.

#### Acceptance Criteria

1. THE system SHALL use Framer Motion library for all animations
2. WHEN a page loads, THE system SHALL animate components with staggered fade-in (100ms delay between elements)
3. WHEN a user hovers over interactive elements, THE system SHALL scale the element to 1.05x within 200ms
4. WHEN a user clicks a button, THE system SHALL provide haptic-style feedback animation (scale down then up)
5. WHEN data updates, THE system SHALL animate the transition using spring physics
6. THE AI_Analysis step SHALL display an animated progress indicator with particle effects
7. WHEN Auto_Mode is enabled, THE toggle SHALL animate with a smooth slide and color transition
8. THE Live_Feed SHALL animate new entries sliding in from the bottom
9. THE Growth_Metrics SHALL count up from previous value to new value over 1 second
10. THE system SHALL reduce motion for users with prefers-reduced-motion preference

### Requirement 8: Results-Oriented Content Display

**User Story:** As a user, I want to see business results instead of technical details, so that I understand the value I'm getting.

#### Acceptance Criteria

1. THE system SHALL display Growth_Metrics as business outcomes ("+₹12,000 revenue" not "Generated 5 posts")
2. THE Live_Feed SHALL describe actions in business terms ("Acquired 3 new leads" not "Posted to LinkedIn")
3. THE AI_Insights SHALL present recommendations as growth opportunities ("Increase revenue by 15%" not "Try this feature")
4. THE Dashboard SHALL prioritize revenue and lead metrics over activity metrics
5. WHEN displaying content generation, THE system SHALL show expected reach and engagement
6. WHEN displaying ad campaigns, THE system SHALL show projected ROI and conversion rate
7. THE system SHALL use currency formatting (₹) for all monetary values
8. THE system SHALL use percentage formatting for growth indicators
9. THE system SHALL display time-based metrics (daily, weekly, monthly) with clear labels
10. THE system SHALL avoid technical jargon in user-facing text

### Requirement 9: One-Click Action Pattern

**User Story:** As a user, I want to accomplish tasks with minimal effort, so that I can act quickly without overthinking.

#### Acceptance Criteria

1. THE Action_Panel SHALL execute actions with a single click (no confirmation dialogs for non-destructive actions)
2. WHEN a user clicks "Generate Content", THE system SHALL create content immediately without additional prompts
3. WHEN a user clicks an AI_Insights recommendation, THE system SHALL execute the action without confirmation
4. THE system SHALL provide inline editing for generated content (click to edit)
5. THE system SHALL auto-save all changes without requiring explicit save actions
6. THE Dashboard SHALL provide quick action buttons in the header (no navigation required)
7. WHEN a user enables Auto_Mode, THE system SHALL activate immediately without configuration steps
8. THE system SHALL use smart defaults for all actions (user can customize later)
9. THE system SHALL display action results inline (no page navigation)
10. THE system SHALL provide undo functionality for destructive actions (delete, disable)

### Requirement 10: Gamification and Engagement

**User Story:** As a user, I want to feel motivated to use the product daily, so that I build a habit and see consistent growth.

#### Acceptance Criteria

1. THE Dashboard SHALL display a Growth_Score calculated from user activity and business results
2. THE Growth_Score SHALL update in real-time as users complete actions
3. THE system SHALL display a streak counter showing consecutive days of activity
4. WHEN a user reaches a milestone (7-day streak, 100 leads), THE system SHALL display a celebration animation
5. THE Dashboard SHALL show progress bars for weekly goals (content generated, leads acquired)
6. THE system SHALL award badges for achievements (first campaign, first ₹10k revenue)
7. THE AI_Insights SHALL include a "Quick Win" recommendation for immediate gratification
8. THE system SHALL display a daily checklist of recommended actions
9. WHEN a user completes the daily checklist, THE system SHALL display a completion animation
10. THE system SHALL send notifications for significant achievements (not spam)

### Requirement 11: Mobile-Responsive Design

**User Story:** As a mobile user, I want full functionality on my phone, so that I can manage my business on the go.

#### Acceptance Criteria

1. THE system SHALL render correctly on screen widths from 320px to 2560px
2. THE Dashboard SHALL use a single-column layout on screens below 768px width
3. THE Action_Panel SHALL stack vertically on mobile devices
4. THE Growth_Metrics SHALL remain visible above the fold on mobile devices
5. THE Live_Feed SHALL be scrollable with touch gestures
6. THE Pricing_Page SHALL display tiers in a vertical stack on mobile devices
7. THE Onboarding_Flow SHALL adapt form layouts for mobile screens
8. THE system SHALL use touch-friendly button sizes (minimum 44x44px)
9. THE system SHALL support swipe gestures for navigation where appropriate
10. THE system SHALL load mobile-optimized images and assets

### Requirement 12: Performance and Loading States

**User Story:** As a user, I want the interface to feel fast and responsive, so that I don't waste time waiting.

#### Acceptance Criteria

1. THE system SHALL display loading skeletons for async content (no blank screens)
2. THE Dashboard SHALL render initial view within 1 second
3. WHEN fetching data, THE system SHALL show progress indicators
4. THE system SHALL implement optimistic UI updates (show result before server confirmation)
5. THE system SHALL cache frequently accessed data in browser storage
6. THE system SHALL lazy-load images and heavy components
7. THE system SHALL prefetch likely next pages (Pricing_Page when viewing Dashboard)
8. THE system SHALL use React Query for efficient data fetching and caching
9. THE system SHALL display error states with retry options
10. THE system SHALL maintain 60fps animation performance on modern devices

### Requirement 13: User Flow Optimization

**User Story:** As a new user, I want a seamless journey from landing to activation, so that I become a paying customer quickly.

#### Acceptance Criteria

1. THE system SHALL implement the flow: Landing_Page → Onboarding_Flow → Dashboard → Pricing_Page
2. THE Landing_Page SHALL include a secondary CTA to view Pricing_Page directly
3. THE Onboarding_Flow SHALL display progress indicators (Step 1 of 3)
4. WHEN a user completes Onboarding_Flow, THE system SHALL display an upgrade prompt if on free tier
5. THE Dashboard SHALL include a persistent upgrade banner for non-paying users
6. WHEN a user enables Auto_Mode without subscription, THE system SHALL navigate to Pricing_Page
7. THE system SHALL remember user position in flow (resume if interrupted)
8. THE system SHALL track conversion funnel metrics (landing → signup → activation → payment)
9. THE system SHALL A/B test critical conversion points (CTA text, pricing display)
10. THE system SHALL minimize friction points (no email verification required for trial)

### Requirement 14: Social Proof and Trust Signals

**User Story:** As a skeptical visitor, I want to see evidence that AstraMark works, so that I trust it with my business.

#### Acceptance Criteria

1. THE Landing_Page SHALL display customer testimonials with photos and business names
2. THE Landing_Page SHALL show aggregate metrics (e.g., "₹50Cr+ revenue generated for customers")
3. THE Landing_Page SHALL include logos of recognizable customer companies
4. THE Pricing_Page SHALL display customer success stories for each tier
5. THE Dashboard SHALL show industry benchmarks ("You're in the top 10% of users")
6. THE system SHALL display real-time activity feed on Landing_Page ("Rajesh just generated ₹5k in leads")
7. THE system SHALL include trust badges (security, privacy, certifications)
8. THE Landing_Page SHALL link to case studies with detailed results
9. THE system SHALL display review ratings from third-party platforms
10. THE system SHALL show money-back guarantee prominently

### Requirement 15: Accessibility and Internationalization

**User Story:** As a user with accessibility needs, I want to use all features comfortably, so that I'm not excluded from the product.

#### Acceptance Criteria

1. THE system SHALL support keyboard navigation for all interactive elements
2. THE system SHALL provide ARIA labels for screen readers
3. THE system SHALL maintain focus indicators with 3px outline
4. THE system SHALL support screen reader announcements for dynamic content updates
5. THE system SHALL allow text scaling up to 200% without breaking layout
6. THE system SHALL provide alt text for all images
7. THE system SHALL use semantic HTML elements (nav, main, article, aside)
8. THE system SHALL support high contrast mode
9. THE system SHALL display currency in Indian Rupees (₹) with proper formatting
10. THE system SHALL use Indian English for all copy (e.g., "₹1,00,000" not "₹100,000")

## Success Metrics

The following metrics will be tracked to measure the success of this implementation:

1. **Conversion Rate**: Percentage of Landing_Page visitors who complete signup
2. **Time to WOW Moment**: Duration from signup to completing Onboarding_Flow
3. **Auto_Mode Adoption**: Percentage of users who enable Auto_Mode
4. **Retention Rate**: Percentage of users active after 7, 30, and 90 days
5. **Upgrade Rate**: Percentage of free users who convert to paid plans
6. **Daily Active Users**: Number of users engaging with Dashboard daily
7. **Growth_Score Average**: Mean Growth_Score across all active users
8. **Page Load Time**: 95th percentile load time for critical pages
9. **Mobile Usage**: Percentage of sessions from mobile devices
10. **Customer Lifetime Value**: Average revenue per user over 12 months

## Technical Constraints

1. The system SHALL be built using React 19.x with functional components and hooks
2. The system SHALL use Tailwind CSS for styling
3. The system SHALL use ShadCN UI components as the base component library
4. The system SHALL use Framer Motion for animations
5. The system SHALL use React Query for data fetching and state management
6. The system SHALL integrate with existing backend API endpoints
7. The system SHALL maintain compatibility with current authentication system
8. The system SHALL support modern browsers (Chrome, Firefox, Safari, Edge - last 2 versions)
9. The system SHALL be deployable as a static site
10. The system SHALL maintain current project structure (Create React App with Craco)

## Out of Scope

The following items are explicitly out of scope for this feature:

1. Backend API changes or new endpoint development
2. Payment gateway integration (assumes existing Razorpay integration)
3. Email service integration (assumes existing email system)
4. AI model training or algorithm changes
5. Mobile native applications (iOS/Android)
6. Multi-language support beyond Indian English
7. White-label or multi-tenant features
8. Advanced analytics dashboard (beyond Growth_Metrics)
9. Team collaboration features
10. API access for third-party integrations
